# 距離モード フィードバック表示の改善

## Goal

- フィードバック図のスケールを固定し、矢印の長さが常に半音距離に比例するようにする。
- 矢印下の半音目盛り（丸・トラック・数値ラベル）を削除する。
- **正解・基準音・回答音のラベルは残し、しかるべき位置に表示する**（日本語表示維持）。
- **矢印の根元は基準音の丸の上に重なる（凌駕）**。日本の矢印で2本並ぶため、**基準音の丸は2つ描画**する（上段・下段レーンに1つずつ）。
- **矢印の先端には丸を描かない**。ラベルは**矢印の上**に表示する。

## 現状

- **スケール**: `buildDistanceFeedbackDiagramSteps` が `maxDistance = max(0, |正解|, |回答|)` で可変。出題範囲が小さいと列数が減り、同じ「3半音」の矢印でも問題ごとに物理的な長さが変わる。
- **矢印下の要素**: 各 step ごとに `__track`（水平バー）、`__marker`（丸）、`__label`（-3, -2 等）を描画。これらは削除対象。
- **ラベル**: 現在は「正解」「解答」のみ。各矢印の位置（step.distance が一致する列）の `annotation-stack` に表示。**基準音（距離 0）のラベルは未実装。**

## 変更方針

### 1. スケールの固定

- 定数 `DISTANCE_FEEDBACK_SCALE_MAX_SEMITONES = 12` を定義し、常に 0..12 の step を生成する。
- 正解/回答は表示用に `Math.min(..., 12)` でクランプ。
- 列数が常に 13 になり、1半音 = 1列分で統一される。

### 2. 矢印下の要素：基準音の丸のみ残す（2つ）、他は削除

- **削除するもの**: 全 step の `__track`（水平バー）と `__label`（-3, -2 等の数値）。矢印**先端**側の丸（正解・回答の位置の `__marker`）は描画しない。
- **残すもの**: **基準音（距離 0）の列だけ**、丸を描画する。日本の矢印は上段・下段の2本並ぶため、**基準音の丸は2つ**（`data-lane="upper"` / `"lower"`）を、矢印レーンに合わせて配置する。
- **矢印の根元が基準音の丸を凌駕**: 矢印レイヤーの z-index を丸より高くし、根元が丸の上に重なるようにする（視覚的に矢印が丸から出ているように見える）。
- `.ui-distance-diagram__step` の `grid-template-rows` は、アノテーション + スペーサー + 基準音列のみ丸用の行を追加（distance === 0 のときのみ2つの `__marker` を表示）。他列では `__track` / `__marker` / `__label` は描画しない。不要になった `__track` / `__label` の CSS は削除してよい。`__marker` は基準音の2つの丸用に残す（lane 用スタイル追加）。

### 3. 正解・基準音・回答音ラベルの維持と配置

- **残す・追加するラベル**
  - **基準音**: 距離 0 の列に「基準音」を表示（新規追加）。
  - **正解**: 正解半音数の列に「正解」を表示（現状どおり）。
  - **回答音**: 回答半音数の列に「回答音」を表示（現状の「解答」を「回答音」に変更する）。
- **実装**
  - [`distance-feedback-annotations.ts`](src/app/train/distance-feedback-annotations.ts): 型を `label: "正解" | "解答" | "基準音"` から `"正解" | "回答音" | "基準音"` に変更。`buildDistanceFeedbackDiagramAnnotations` で (1) 基準音用に `distance: 0, label: "基準音", tone: "neutral"` を追加、(2) 回答側のラベルを「解答」→「回答音」に変更。
  - ソート順: 0（基準音）、正解、回答音の順で並ぶようにする（既存の sort を拡張）。
  - [`globals.css`](src/app/globals.css): `.ui-distance-diagram__annotation[data-tone="neutral"]` を追加し、基準音ラベル用のスタイル（枠・文字色）を定義。既存の neutral 系（例: `--border-strong`）に合わせる。
- **位置**: 既存と同様、各 step の `annotation-stack` に、`annotation.distance === step.distance` のアノテーションだけが表示される。したがって基準音は 0 の列、正解は正解の列、回答音は回答の列に自動的に配置される。**ラベルは矢印の上**（アノテーション行）に表示するレイアウトを維持する。

## 検証

- `npm run verify`
- 距離モードでフィードバック画面を目視: 矢印長が半音距離に比例、**基準音の丸が2つ（根元）のみ**・矢印先端に丸なし・目盛りバー・数値なし、**矢印の根元が基準音の丸の上に重なる**、**ラベル（基準音・正解・回答音）が矢印の上に正しい列で表示されていること**
- 既存 Stories（正解/解答の表示確認）を「正解/回答音」に合わせて期待値更新

## 補足

- 日本語表示は維持。最大半音数は 12 固定（config は渡さない）。
