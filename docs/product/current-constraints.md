# current-constraints

最終更新: 2026-03-11

## 現在の固定方針
- MVP は Web アプリとして実装する
- スマホブラウザ縦向き利用を最優先とする
- 認証は Better Auth + Google OAuth を採用する
- DB は PostgreSQL + Drizzle ORM を採用する
- 音源は Web Audio API でクライアント側生成・再生する
- 問題出題中・回答中は DB にアクセスしない
- セッション結果は終了時にまとめて保存する
- ゲスト利用では結果保存しない
- train 画面は 1 ルート内の状態切り替えで構成する
- スコアは内部計算・DB保存ともに小数で扱い、小数第3位まで保持する
- 表示時のみ必要に応じて丸める
- UI と純粋ロジックは分離する
- 純粋ロジックは `features/training/model` に寄せる
- MVP では不要な REST API を増やさない

## 実装上の補足
- training 中の進行状態はクライアント側で保持する
- 保存はログイン時のみクラウド保存を前提とする
- 設定は即時反映を重視しつつ、保存は適切なタイミングで行う
- 学習テンポを崩す演出や重い処理は避ける
